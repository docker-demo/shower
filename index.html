What is docker?

A tool designed to make it easier to create, deploy, and run applications by using containers.

What is a container?

  1. It's not a Virtual Machine
  2. But it feels like a virtual machine
  3. VMs virtualize hardware for each guest
  4. Containers all share the Host Kernel
  5. Containers are isolated from each other like VMs
  6. Containers are smaller than VMs

Docker concepts/components

  • Docker Host
  • Docker Engine (Docker Daemon)
  • Docker Client
  • Docker Containers
  • Docker Images and Dockerfile
  • Docker Registry and Docker Hub

Docker Benefits

  1. Consistent runtime environments
  2. Lightweight deployments
  3. Isolation

Let's Play

Play with docker https://labs.play-with-docker.com/

			docker run hello-world

What just happened?

  1. Docker pulled the official image of hello-world from docker.com
  2. Then it created a container from the image, attached stdout from your terminal
  3. And started the container
  4. Hello world ran, printed stuff and exited, leaving you where you started
				docker run docker/whalesay cowsay hello
			
			docker run -it alpine
  1. Alpint is a very lightweight linux
  2. The -it option allocated a psuedoTTY and attached STDIN which makes the container interactive
  3. Try running vi
  4. exit
